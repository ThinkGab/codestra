**2c — pushToWorker helper (add in the Helpers section, after the generateId function at line 68, before the Routes comment at line 70):**

```js
async function pushToWorker(worker, msg) {
  if (!worker.callback_url) return false;
  try {
    const res = await fetch(worker.callback_url, {
      method: "POST",
      headers: {
        "Content-Type": "application/json",
        "Authorization": `Bearer ${SECRET}`,
      },
      body: JSON.stringify({
        id: msg.id,
        from: msg.from,
        to: msg.to,
        body: msg.body,
        timestamp: msg.timestamp,
      }),
      signal: AbortSignal.timeout(5000),
    });
    return res.ok; // true on 2xx
  } catch {
    return false;
  }
}
```

Per D-04: push payload is `{id, from, to, body, timestamp}` — the full message object minus `readBy` (which is internal hub state).
Per D-05: any error (network, timeout, non-2xx) is swallowed — returns false.
Per D-08: Authorization header uses the same Bearer SECRET pattern as hub's own auth.
Per D-09: AbortSignal.timeout(5000) gives a 5-second hard timeout per push (Node 20+ built-in, no new dependencies).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| worker→hub (POST /workers) | Worker supplies callback_url; hub stores it without fetching it at registration time |
| hub→worker (pushToWorker) | Hub makes outbound HTTP request to an attacker-controlled URL if callback_url is poisoned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Spoofing | pushToWorker Authorization header | mitigate | Forward SWARM_SECRET as Bearer token — worker validates same secret (per D-08) |
| T-03-02 | Tampering | callback_url field (POST /workers) | accept | Hub is LAN-only, all workers share SWARM_SECRET auth; SSRF risk accepted for trusted LAN scope |
| T-03-03 | Denial of Service | pushToWorker fetch hang | mitigate | AbortSignal.timeout(5000) caps each push at 5 s (per D-09) |
| T-03-04 | Information Disclosure | readBy Set in JSON responses | accept | readBy is internal hub state; not serialized in GET /messages response (only matching array returned) |
</threat_model>
